## Objective

Benchmark local coding models running on a MacBook Pro M3 Max (48 GB) against each
other, cloud models accessed through OpenRouter (e.g. GLM, Kimi K2), and Codex CLI
agent runs. The aim is to identify the fastest and most capable local coding setup,
quantify the gap to frontier cloud models, and measure Codex as a first-class
coding-agent baseline.

## Tech Stack

- **Language**: Python (managed with `uv`)
- **Framework**: None
- **Runtime**: CPython 3.x

## Architecture

CLI benchmark harness. Talks to any OpenAI-compatible `/v1/chat/completions`
endpoint so the same code measures local MLX servers and OpenRouter by swapping the
base URL + API key. Measures per-turn time-to-first-token (prefill tok/s), decode
tok/s, and total latency.

## Notes

- **Fixed hardware**: MacBook Pro M3 Max, 48 GB unified memory. Reference articles
  used M4 64 GB — the 48 GB ceiling constrains which quantized models co-resident
  (target + draft/KV cache).
- **Core design constraint**: one harness, swappable base URL — local serving
  (`mlx_lm.server`, `ollama serve`) vs OpenRouter — both
  OpenAI-compatible. Serving runtime to be settled experimentally.
- **Candidate roster discussed** (to refine in `/brainstorm`): local Qwen3.6 dense
  (4-bit, via `mlx-lm`) and Qwen3.6-35B-A3B MoE (via
  `ollama`); cloud GLM and Kimi K2 via OpenRouter; Codex CLI as agent baseline.
- **Methodology reference** (`articles/`): two-part Medium series by Manjunath
  Janardhan. Part 1 = local Claude Code setup via `claude-code-router` (CCR) bridge.
  Part 2 = benchmark harness that bypasses the agent loop, hits the endpoint
  directly, and measures TTFT/prefill, decode, latency across **task mode** (focused
  sub-prompts) and **sweep mode** (preamble padded to 2k/8k/16k/24k tokens). Key
  finding: agentic coding is prefill-bound, not decode-bound.
- **Open questions for `/brainstorm`**: quality scoring (not just speed), which
  coding task suite to standardize on, how to fairly normalize cloud vs local cost,
  and how to drive Codex reproducibly through `codex exec`.
